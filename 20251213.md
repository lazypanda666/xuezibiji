# 统计学习方法

## 1. 概念

### （1）对象

* **数据**
* 前提：同类数据具有一定的统计**规律性**

### （2）目的

对数据进行**预测**和**分析**

### （3）方法

三要素：**模型**、**策略**、**算法**

步骤：

* 得到一个有限的训练数据集合
* 确定包含所有可能的模型的假设空间（学习模型的**集合**）
* 确定模型选择的准则（学习的**策略**）
* 实现求解最优模型的算法（学习的**算法**）
* 通过学习方法选择**最优**模型
* 利用学习的最优模型对新数据进行预测或分析

### （4）研究

* 统计学习**方法**
* 统计学习**理论**
* 统计学习**应用**

## 2. 分类

### （1）基本分类

#### A. 监督学习

##### **定义**：

从**标注数据**（即已知输入输出对应关系的数据）中学习预测模型的机器学习问题。其本质是学习输入到输出的映射的统计规律

##### **关键概念**：

- **输入空间、特征空间与输出空间**：

  * 输入所有可能取值的集合称为**输入空间**

  * 每个**输入实例**通常用**特征向量**表示，所有特征向量构成**特征空间**。模型定义在特征空间上

  * 输出所有可能取值的集合称为**输出空间**

  * 输入与输出对又称为**样本**或**样本点**

- **问题的类型**：根据输入(X)和输出(Y)变量的类型不同，监督学习任务主要分为：

  - **回归问题**：输入变量和输出变量为连续变量
  - **分类问题**：输出变量为有限个离散变量
  - **标注问题**：输入变量和输出变量为变量序列

- **联合概率分布**：

  监督学习的基本假设是：输入与输出的随机变量 **X** 和 **Y** 遵循一个**未知的联合概率分布 P(X, Y)**。训练数据和测试数据被视作从该分布中独立同分布采样产生的。

- **假设空间**：

  模型（即输入到输出的映射）的集合称为**假设空间**。监督学习的目标就是从该空间中寻找“最优”模型。
  
  模型可以表示为：
  
  - 条件概率分布：`P(Y|X)`
  - 决策函数：`Y = f(X)`

**流程**   

- **学习过程**： 
  * **输入**：训练数据集 `T = {(x1, y1), (x2, y2), ..., (xN, yN)}`
  * **目标**：通过学习算法，从假设空间中学习得到一个模型，表示为 `Ŷ = f̂(X)`或 `P̂(Y|X)`

- **预测过程**：
      * **输入**：新的测试样本输入 `x_{N+1}`
      * **输出**：模型给出对应的预测输出 `y_{N+1} = f̂(x_{N+1})`或 `y_{N+1} = argmax_y P̂(y|x_{N+1})`

#### B. 无监督学习

##### **定义**：

从**无标注数据**（自然得到的数据）中学习预测模型的机器学习问题，其本质是学习数据中的统计规律或潜在结构

##### 关键概念

- **目标**：模型用于表示数据的**类别**（聚类）、**转换**（降维）或**概率**估计。

- **模型形式**：要学习的模型可以表示为：

  - 函数 `z = g(x)`

  - 条件概率分布 `P(z|x)`或 `P(x|z)`

    （其中 `x`属于输入空间，`z`属于输出的隐式结构空间，如类别标签、降维后的编码等）

- **假设空间**：所有可能模型的集合，无监督学习旨在从中选出在特定评价标准下的最优模型

##### 流程

- **学习过程**：利用无标注训练数据集 `U = {x1, x2, ..., xN}`学习得到模型（如 `z = ĝ(x)`或 `P̂(z|x)`）

- **预测/分析过程**：对新的输入数据或已有数据，使用学习到的模型进行分析（如聚类、降维）或预测

# latex

## 1. 概述

### （1）TEX

TEX 是**高德纳(Donald E. Knuth)** 为**排版文字**和**数学公式**而开发的软件

### （2）latex

LATEX 是一种使用 **TEX 程序**作为排版引擎的格式（format），最初的设计目标是**分离内容与格式**，以便作者能够专注于内容创作而非版式设计，并能以此得到高质量排版的作品。

### （3）latex的优缺点

- 优点：
      -  具有**专业的排版输出**能力，产生的文档看上去就像“印刷品”一样
      -  具有方便而强大的**数学公式排版**能力
      -  绝大多数时候，用户只需专注于一些组织文档结构的基础命令，很少操心文档版面设计
      -  很容易生成复杂的**专业排版元素**，如脚注、交叉引用、参考文献、目录等
      -  强大的**可扩展性**。各地的人开发了数以千计的LATEX宏包用于补充和扩展LATEX的功能
      -  能够促使用户写出**结构良好**的文档
      -  LATEX 和 TEX 及相关软件是**跨平台、免费、开源**的。

- 缺点：
      -  入门**门槛高**
      -  **不容易排查错误**。LATEX作为一个依靠编写代码工作的排版工具，其使用的宏语言比C++ 或Python 等程序设计语言在错误排查方面困难得多。它虽然能够提示错误，但不提供调 试的机制，有时错误提示还很难理解。 
      -  **不容易定制样式**。LATEX 提供了一个基本上良好的样式，为了让用户不去关注样式而专注 于文档结构。但如果想要改进LATEX生成的文档样式则是十分困难
      -   相比“所见即所得”的模式有一些不便，为了查看生成文档的效果，用户总要**不停地编译**。

### （4）命令行基础

- 在命令行中运行程序时，需要先输入**程序名**，其后可加一系列用**空格分隔的参数**，并按下 Enter 键执行。

- 一般情况下，命令行程序执行完毕会**自行退出**。若遇到错误或中断，可输入**Ctrl+C** 以强制结束。 使用命令行程序输入、输出文件时，需确保**文件路径正确**。通常需要先切换到文件所在目录， 再执行有关程序。切换路径可以执行

```c
cd <path>
```

- 注意⟨path⟩ 中的多级目录在**Windows** 系统上使用<u>反斜线\</u>分隔，而在**Lunix**系统上使用<u>正斜线/</u>分隔。如果⟨path⟩ 中带有**空格**，则需加上引号 " 。此外，在Windows系统上如果要切换到其他分区，还需加上/d选项，例如cd /d "C:\Program Files (x86)\"
